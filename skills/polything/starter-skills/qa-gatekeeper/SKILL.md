# qa-gatekeeper

## When to use
- Before publishing anything externally (landing page, email, deck, ad copy).

## Inputs
- Any draft artefact (paste it or point to a file)
- Optional: outputs/client_brief.md

## Outputs
- outputs/qa_report.md

## Instructions
1. Apply templates/qa_rubric.md as a checklist.
2. Mark PASS/FAIL.
3. Provide a short fix list with direct edits.
4. If FAIL, suggest the smallest changes needed to reach PASS.
